# Architecture Decision Records

Use this folder for durable, numbered decisions that affect system structure, operating constraints, or long-lived contracts.

## Rules

- One decision per file
- Keep records append-only; supersede instead of rewriting history
- Use the template in `template.md`
- Reference ADR numbers from architecture and spec docs when relevant
